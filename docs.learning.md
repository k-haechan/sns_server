@Column(length = …) 의미와 실제 동작
1.	JPA 레벨에서의 의미
•	@Column(length = 50)는 JPA가 문자열 길이 제한을 DB 컬럼에 매핑할 때 참고하는 정보입니다.
•	예를 들어, DDL을 자동 생성(hibernate.ddl-auto)할 때 컬럼 타입의 최대 길이를 지정합니다.
2.	실제 DB에서 컬럼 길이와 충돌 시
•	만약 DB에 이미 VARCHAR(100)로 정의되어 있다면, @Column(length = 50)는 JPA에서만 참고할 뿐, DB는 여전히 100글자를 허용합니다.
•	따라서 51~100글자는 DB 제약에서는 문제가 없고, JPA 제약으로도 강제되지 않습니다.
3.	따라서 유의할 점
•	DB 스키마와 JPA 매핑의 일관성을 유지해야 합니다.
•	실제 제약을 DB 레벨에서 관리하려면 Flyway 등의 마이그레이션 파일에서 VARCHAR(50)처럼 정의해야 합니다.
